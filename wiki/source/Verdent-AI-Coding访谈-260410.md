# 对话 Verdent AI：web coding 初创公司

> 来源：访谈 | 日期：2026-04-10
> 核心标的：Verdent AI（AI coding 初创）

## 来源概要

对 Verdent AI（估值 $2 亿，腾讯和红杉领投）算法工程 VP 的访谈，覆盖公司发展历程、产品策略、与大厂的竞争关系、AI coding 赛道趋势。

## 关键要点

1. **24 年 12 月起步，三人团队**：国内当时唯一独立 AI coding 创业公司，现有 10 万海外开发者
2. **不做中国市场**：因美国政策限制，无法对中国/香港/俄罗斯等非白名单国家提供服务
3. **插件先行策略**：IDE 插件获客成本低，能装进 Cursor、Trae 等竞品生态
4. **大厂部门墙是机会**：大厂做基模和做应用是两个事业群，内部调模型和外部 API 调用没有差异
5. **核心理念**：把 vibe coding 最佳实践沉淀成产品功能
6. **代码 review 是当前痛点**：多模型交叉审查——"让魔法打败魔法"
7. **生存框架**："站在模型延长线上"——模型未来能学会的少做，训不进模型的多做
8. **提前半年做功能**：等模型能力追上来再释放

## 值得注意的观点/引语

> 大厂做基模和做应用是两个事业群，内部调模型和外部人去调，没有任何差异。他们不会为你的产品做优化。

> 有些事情要想一想模型未来会做哪些方面的训练，那些事就少做。有些事情是怎么样都训不到模型里的，那就多做。

> 我们会提前半年做一个功能出来，然后等半年之后再放出来。因为功能做好了，当时模型的效果还达不到。

## 结构性观察

- **大厂部门墙（基模团队 vs 应用团队分离）是创业公司的结构性机会，而非偶然的组织疏漏**：大厂内部调模型与外部 API 调用"没有任何差异"——这意味着大厂的应用团队对于优化模型能力并没有独特的内部优势，创业公司通过快速迭代反而能比大厂内部团队更敏锐地捕捉模型能力的边界。这一洞察直接影响 AI 应用层的竞争格局预判。
- **"站在模型延长线上"的产品策略，是 AI 应用创业公司的生存基础**：不做模型未来会"自动学会"的功能，只做"训不进模型"的工程复杂性和用户体验——这是一种高度依赖模型能力预判的元策略。对投资者而言，这意味着优秀的 AI 应用创业公司本质上是"模型能力的预测交易者"，其产品价值与模型进化速度深度绑定。
- **"提前半年做功能、等模型能力追上再发布"揭示了 AI 应用开发的独特时间结构**：功能研发和产品发布之间有半年的能力等待期——这不同于传统软件（做完即发），也不同于平台软件（依赖用户数据）。这对评估 AI 应用公司的研发投入和产品路线图有直接影响：当期在研功能的价值要用半年后的模型能力来估算。
- **国内 AI coding 创业公司不做中国市场（美国政策限制），反而以此为契机专注海外开发者**：这一看似被动的约束，实际上使 Verdent 规避了与字节/腾讯/百度直接竞争，也使其产品迭代更快贴合海外（主要是西方）开发者习惯——是监管约束倒逼差异化定位的典型案例。
