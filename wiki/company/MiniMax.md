# MiniMax

## 公司概况

MiniMax 是国内头部垂直大模型公司，早期以 C 端产品（陪伴类应用、海螺视频）起家，后转向开放平台和 ToB 业务，以极致性价比路线在 Agent/Coding 场景取得市场红利。2026 年以来正将战略重心重新调整回多模态领域。

（来源：[[source/ace-头部AI模型竞争格局-260413]]）

## 商业模式

- API 对外服务（极致性价比定位）
- To B 业务（Agent/Coding 场景）
- 早期 C 端产品：Character.ai 对标产品（陪伴类）、海螺视频（文生视频）

## 技术架构

### 参数规模

- **总参数量**：约 230B（2300 亿），是 Kimi 的约 1/4，智谱的约 1/3
- **活跃参数**：约 10B（100 亿），是 Kimi 的约 1/3，智谱的约 1/4
- M2 系列（M2.1/M2.5/M2.7）：主要针对 Coding 场景进行强化学习

### 性价比策略

- 通过超高缓存命中率（实际推理中 >90%）将输入 token 实际价格压至远低于标价
- 在参数量上作出明确取舍（1/3-1/4 于竞争对手），换取最低推理成本
- 策略定位：极致性价比路线，"偏科"（聚焦 Coding/Agent）

## 战略演进

### 历史路径

1. **2021-2024**：产品驱动期，C 端应用（陪伴类 + 海螺视频）
2. **2025**：转向技术驱动，抓住 Agent/Coding 市场红利，以 M 系列模型获取 B2B 开放平台机会
3. **2026 年以来**：战略重心重新转向多模态，准备整合视频+语音+文本的原生全模态模型

### 多模态战略（当前重心）

（来源：[[source/ace-头部AI模型竞争格局-260413]]）

- 国内拥有顶尖视频模型之一（海螺视频）+ 优秀语音模型能力
- 计划将视频和语音两个模态优势融合进统一多模态大模型
- 目标：类比 Gemini 多模态领先逻辑，形成差异化优势
- M3 模型（布局中）：多模态能力积累强于 Kimi/智谱
- 截至 2026 年 4 月，Kimi 2.5 支持图像输入，但 MiniMax 计划做到视频模态输入的完整多模态

### 放弃 Coding 赛道的理由

- Coding Agent 赛道极度拥挤，Anthropic Claude 已占据主导
- 产品力不如 Kimi，定制化交付不如智谱
- 资源有限，需要聚焦差异化

## 算力约束

- GPU 严重短缺，是最主要的增长约束
- 与字节跳动的竞争区域重叠（视频生成）使 GPU 竞争更激烈
- 缺少私域视频数据（无 YouTube/抖音/快手类平台），视频模型竞争力受限

## 竞争格局

| 指标 | MiniMax | Kimi | 智谱 |
|------|---------|------|------|
| 总参数 | ~230B | ~1T | ~700B |
| 活跃参数 | ~10B | ~30B | ~40B |
| 核心优势 | 极致性价比 | 长上下文/Agent | Coding/B2B |
| 当前弱点 | 绝对能力不是最强 | — | 算力约束 |
| 战略重心 | 多模态 | Agent/Coding | B2B Coding |

## 风险因素

- GPU 短缺限制扩张（视频/多模态耗算力最高）
- C 端业务（Character.ai 对标、海螺视频）遇到增长瓶颈
- 多模态战略是否能在 Kimi/Gemini 面前建立真正差异化，有不确定性

## 催化剂

- M3 多模态模型发布：实现视频+语音+文本原生全模态
- ToB Agent 市场爆发：长尾非关键任务场景性价比优势凸显
- 算力获取突破
