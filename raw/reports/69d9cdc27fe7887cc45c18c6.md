# 260410 - 对话Verdent AI（web coding初创公司）算法工程VP

# 要点

24年12月起步，三人团队，国内当时唯一做AI coding的独立创业公司，现在10万海外开发者
插件先行策略：IDE插件获客成本低，能装进Cursor、Trae等竞品生态，借别人的流量长自己的用户
大厂部门墙：做基模和做应用是两个事业群，内部调模型和外部API调用没有任何差异，创业公司反而更灵活
产品核心理念：把自己vibe coding的最佳实践沉淀成产品功能
阶段性解决痛点：早期是规划（需求→技术文档），现在是代码review（多模型交叉审查，"让魔法打败魔法"）
生存框架："站在模型延长线上"——模型未来能学会的少做，训不进模型的多做
反直觉操作：提前半年做好功能，等模型能力追上来再释放

# 专家观点

Verdent AI是一家由前字节跳动算法负责人陈志杰与前百度技术与产品负责人刘晓春创立的AI公司，目前由腾讯和红杉资本领投，估值2亿美元。
V = Verdent AI VP，E = Ellen。
E：你们什么时候开始做的？
V：24 年 12 月正式开始，25 年 1 月公司成立，当时三个人。
E：最早是从插件开始的？
V：对，IDE 里面的插件，不是 Chrome 插件。我们的插件能装在所有 hook VSCode 的产品里——Cursor、Trae、Kudo 都能装，因为它们 hook 了 VSCode。插件安装成本低，用户不需要改变使用习惯就可以用。25 年的时候先做插件，积累用户，因为插件是一个获客成本很低的东西。
E：后来怎么开始做 web coding 的？
V：差不多六七月份，五月份 Claude Code 出来了，出来之后没多久我们就开始做。也不算转型，就是人多了之后会想一些新的东西。我们看到了模型背后更大的能力，觉得未来的趋势就是有些事情可以让 agent 自己来做。
E：中间迭代了几版，思路是什么？
V：本质上没什么大变化，就是改样式、修 bug、加新功能。比如 MCP 刚出来的时候、后来出了 Skill，每次业内有新的技术点出来，就会把它产品化。我们迭代产品就一条核心理念：把我们在 vibe coding 中的最佳实践沉淀成产品。有一些变成显性的功能——按钮、输入框，但大部分变成了 agent 的内在能力。
E：从去年到现在，有哪些想法没变过？
V：最大的不变是整体 UI 交互。我们还是把对话框作为主视觉，就像 ChatGPT 一样，对话页面里穿插工具的执行结果。我们最近在做网页版，比如我现在在车上就可以给云端发消息，让它去写代码干活。
E：国内做 AI native coding 的创业公司多吗？
V：我们刚开始创业的时候并不多，24 年 12 月国内创业公司就我们一家。那些做这个的都是大厂——阿里、字节。字节做的早一点，阿里后面才做的多。创业公司做的比较少。
E：这个赛道不应该是大厂或者大模型公司来做吗？
V：对，但我们之前就在大厂。大厂会有部门墙——做模型的团队和做应用的是两个团队，甚至是两个事业群。做基模的和做业务的，大概是什么效果呢？他们调自己的模型和我们外部人去调，没有任何差异。 他们不会为你的产品做优化。虽然是一个公司、一个大的 team，但除了结算走内部费用之外，其他时候完全是两个没什么相关的团队。
E：你们现在有多少用户？
V：开发者有 10 万人。
E：中国为主吗？
V：不是，我们不做中国。中国如果不挂代理，用不了我们产品上的海外模型。因为美国政策的影响，我们不能对中国、香港、俄罗斯等非白名单国家提供服务。
E：大厂抄你们容易吗？
V：容不容易看他们自己的判断。大厂比较慢，决策慢，而且大厂有自己的想法，未必会抄一个创业公司的想法。虽然也在研究我们。
E：你们的 killer feature 是什么？
V：每个阶段不一样。我们一直在解决每个阶段最大的痛点。
最开始解决的是规划——用户的 prompt 不会给模型，其实是一个文档化的过程，把需求变成技术文档再给模型，比手敲几百行字快很多。第一个产品叫 Dream。
最近的痛点是代码 review——每天写了几千行代码，谁来保证质量？Claude 告诉你写完了，它说的就是对的吗？有的用户不懂代码，或者写的是不精通的语言，怎么办？做法是让 GPT 做 review，或者让 Gemini、GPT 一起去 review，多个模型交叉看代码，快速收敛问题。让魔法打败魔法。
E：每次模型更新你们怎么跟得上？
V：第一，做的事情尽量站在模型的延长线上。有些功能你做完之后，下一版模型出来效果自然变好——这种事尽量少做。虽然能领先当前同类产品，但它是脏活累活，模型变聪明之后就不会犯这种蠢事了。
有些事情要想一想模型未来会做哪些方面的训练，那些事就少做。有些事情是怎么样都训不到模型里的，那就多做。我们的算法团队来自大厂的大模型专家，他们有相应的判断力。
E：你们会跟着模型能力的发展节奏走？
V：对，创业公司最大的点就是要始终跟着模型的能力不断拓展场景。甚至我们会提前半年做一个功能出来，然后等半年之后再放出来。因为功能做好了，当时模型的效果还达不到。但半年之后模型效果到了，你要立马放出来，不然来不及。
E：什么 use case 最受欢迎？
V：没有特别固定的，不同地区有不同的受众。我个人感觉最好的 case 是面对面的时候，你现场出一个需求，我给你演示，半个小时到一个小时一步步做出来。
E：什么样的产品能用你们做，什么不能？
V：几乎所有软件都可以。不能做的：跟硬件强相关的，比如无人驾驶、机器人这种依赖边缘计算的；或者游戏开发，需要专业游戏引擎，它有自己一套闭环开发工具。除此之外，移动端网页、网站、PC 桌面应用、手机 APP、网页产品，全都可以。
